Project 6 - Univinity

Univinity is an iOS app that helps students discover, compare, and track colleges based on their personal preferences. Users can swipe through schools, save favorites, add notes, compare options, and track their application progress all in one place.

Time spent: 12 hours spent in total

Required Features

The following required functionality is completed:

 [x]Users can browse colleges through a swipe-based Discover screen
 [x]Users can save colleges and view them in a Saved tab
 [x]Users can search and filter colleges by criteria (type, size, tuition, etc.)
 [x]Users can view detailed information about each college
 [x]Data is persisted using a backend (Parse)

Optional Features

The following optional features are implemented:

 [x]Favorites system (star colleges and view in a dedicated tab)
 [x]Notes feature for saved colleges
 [x]Compare feature for up to 3 colleges
 [x]Map view to visualize college locations
 [x]Application tracking system with status dropdown (Not Started, Started, Applied, Reviewed)
 [x]Profile preferences to personalize results (GPA, budget, state, major, etc.)
 [x]Onboarding flow with walkthrough + preference survey
 [x]Custom UI styling and animations
Additional Features

The following additional features are implemented:

 [x]Swipe actions with undo, like, and reject buttons
 [x]Toggleable action buttons (via filters)
 [x]Real-time syncing between Saved, Favorites, and Applications
 [x]Match score system based on user preferences
 [x]Custom tab bar UI enhancements
 [x]Improved mobile-friendly layout and spacing

Notes

Challenges encountered while building the app:

Managing asynchronous data fetching from Parse and ensuring consistent UI updates
Preventing list reordering and data inconsistencies when switching tabs
Syncing shared state across multiple views (Saved, Favorites, Applications)
Designing a clean and intuitive UI with multiple features without clutter
Implementing onboarding with user-specific persistence instead of device-based
