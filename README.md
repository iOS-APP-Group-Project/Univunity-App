# Project 6 - *Univinity*

Submitted by: **Sara Kanu**

**Univinity** is an iOS application that helps students discover, evaluate, and track colleges based on their personal preferences. The app allows users to swipe through colleges, save and favorite schools, compare options, add notes, and track application progress in one centralized platform.

Time spent: **12 hours** spent in total

---

## Required Features

The following **required** functionality is completed:

- [x] Users can browse colleges through a swipe-based Discover screen  
- [x] Users can save colleges and view them in a Saved tab  
- [x] Users can search and filter colleges by criteria (type, size, tuition, etc.)  
- [x] Users can view detailed information about each college  
- [x] App data is persisted using a backend (Parse)  

---

## Optional Features

The following **optional** features are implemented:

- [x] Favorites system (star colleges and view in a dedicated tab)  
- [x] Notes feature for saved colleges  
- [x] Compare feature for up to 3 colleges  
- [x] Map view to visualize college locations  
- [x] Application tracking system with status dropdown (Not Started, Started, Applied, Reviewed)  
- [x] Profile preferences to personalize results (GPA, budget, state, major, etc.)  
- [x] Onboarding flow with walkthrough and preference survey  
- [x] Enhanced UI styling and animations  

---

## Additional Features

The following **additional** features are implemented:

- [x] Swipe actions with undo, like, and reject buttons  
- [x] Toggleable action buttons through filters  
- [x] Real-time syncing between Saved, Favorites, and Applications tabs  
- [x] Match score system based on user preferences  
- [x] Improved responsive layout for mobile devices  
- [x] Per-user onboarding (shown on signup instead of per device)

## Notes

Challenges encountered while building the app:

- Handling asynchronous data fetching from Parse and ensuring consistent UI updates  
- Preventing list reordering and inconsistent data when switching tabs  
- Managing shared state across multiple views (Saved, Favorites, Applications)  
- Designing a clean UI while integrating multiple advanced features  
- Implementing onboarding tied to user accounts rather than device storage
